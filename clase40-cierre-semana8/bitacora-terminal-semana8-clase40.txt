Entorno usado: codespaces

1. ¿Qué aprendí sobre el sistema operativo como base de una plataforma?
Respuesta: Aprendí que el sistema operativo es el piso sobre el que se construye todo: gestiona 
hardware, procesos, usuarios y seguridad, y permite que aplicaciones como Node.js y Telegram convivan 
sin conflictos.

2. ¿Qué comandos de terminal usé durante la semana?
Respuesta: Usé comandos como ls, pwd, cd, chmod, chown, ps, top, kill, git status, git add, git commit 
y git push para explorar el sistema, controlar permisos y documentar mi trabajo en el repositorio.

3. ¿Qué aprendí sobre usuarios, grupos y permisos?
Respuesta: Aprendí que cada archivo y proceso tiene dueño y grupo, y que los permisos definen quién puede leer, 
escribir o ejecutar. Esto es clave para proteger recursos y separar responsabilidades.

4. ¿Qué significa mínimo privilegio?
Respuesta: Significa dar a cada usuario o aplicación solo los permisos estrictamente necesarios para su tarea,
evitando accesos innecesarios que puedan convertirse en riesgos.

5. ¿Por qué un permiso como 777 puede ser riesgoso?
Respuesta: Porque permite a cualquier usuario leer, modificar o ejecutar un archivo, lo que expone el sistema 
a cambios no autorizados, ejecución de código malicioso y pérdida de integridad.

6. ¿Qué aprendí sobre procesos?
Respuesta: Aprendí que un proceso es una instancia activa de un programa, que consume recursos y que puede 
estar en estados como ejecución, espera o detenido.

7. ¿Qué puede pasar si un proceso se detiene?
Respuesta: Si un proceso se detiene, el servicio que brinda deja de funcionar, pueden perderse datos en memoria 
y otros procesos dependientes pueden fallar.

8. ¿Qué aprendí sobre logs?
Respuesta: Aprendí que los logs registran eventos del sistema y de aplicaciones, y que son una memoria téc
nica que permite entender qué pasó cuando ocurre un problema.

9. ¿Cómo ayudan los logs al diagnóstico?
Respuesta: Ayudan mostrando errores, advertencias y detalles de ejecución, lo que permite rastrear fallos, 
identificar causas y tomar decisiones para resolver problemas.

10. ¿Qué aprendí sobre Git y trazabilidad?
Respuesta: Aprendí que Git permite llevar un historial ordenado de cambios, con autoría y mensajes claros, 
lo que hace posible reconstruir decisiones y saber quién hizo qué y cuándo.

11. ¿Qué aprendí sobre archivos .env y tokens?
Respuesta: Aprendí que los archivos .env guardan variables sensibles como tokens, y que deben protegerse 
y nunca subirse a repositorios públicos para evitar fugas de credenciales.

12. ¿Cómo puede servir Telegram a una comunidad digital?
Respuesta: Telegram puede servir como canal de comunicación rápido y colaborativo, entregando alertas, 
mensajes de soporte y actualizaciones de proyectos en tiempo real.

13. ¿Qué cuidados debo tener antes de integrar Telegram con Node.js o n8n?
Respuesta: Debo asegurar credenciales, usar variables de entorno, validar entradas, limitar permisos de bots 
y revisar que no se exponga información privada.

14. ¿Qué fue lo más importante de la Semana 8?
Respuesta: Lo más importante fue comprender cómo trabajar con seguridad en Linux y documentar mis aprendizajes 
con evidencia clara, especialmente en la gestión de permisos, procesos y trazabilidad.
